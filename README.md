# Encoding Converter - 文件编码转换插件

一个强大的 VS Code 插件，用于自动检测和转换文件编码，解决中文乱码问题。

## 功能特性

- ✨ **自动检测编码**：智能识别文件的原始编码格式
- 🎯 **优先推荐**：自动检测的编码会在选择列表中置顶并标记
- 🌏 **广泛支持**：支持 GBK, GB2312, Big5, Shift-JIS, EUC-KR 等多种编码
- 🔄 **一键转换**：快速将文件转换为 UTF-8 编码
- 🛡️ **安全检查**：转换前检测文件是否已是 UTF-8 编码

## 使用方法

### 方式一：命令面板
1. 打开需要转换的文件
2. 按 `Ctrl+Shift+P` (Windows/Linux) 或 `Cmd+Shift+P` (Mac) 打开命令面板
3. 输入 `Convert File Encoding to UTF-8` 或 `转换文件编码为UTF-8`
4. 从列表中选择文件的原始编码（自动检测的编码会显示为 ⭐ 标记）
5. 确认后自动转换并重新打开文件

### 方式二：右键菜单
1. 在编辑器或文件浏览器中右键点击文件
2. 选择 `Convert File Encoding to UTF-8`
3. 选择原始编码并确认

## 支持的编码格式

- **简体中文**：GBK, GB2312
- **繁体中文**：Big5
- **日文**：Shift-JIS
- **韩文**：EUC-KR
- **西欧语言**：ISO-8859-1, Windows-1252
- **Unicode**：UTF-8, UTF-16LE, UTF-16BE

## 常见使用场景

### 场景一：打开老旧的 GBK 编码文件显示乱码
```
// 转换前（GBK 编码显示为乱码）
鎴戞槸涓枃

// 转换后（UTF-8 编码正常显示）
我是中文
```

### 场景二：从其他系统迁移代码
当从 Windows 环境迁移代码到 Mac/Linux 时，可能会遇到编码问题。使用本插件可以快速批量转换。

## 安装

### 从 VSIX 文件安装（开发版本）
1. 下载 `.vsix` 文件
2. 打开 VS Code
3. 按 `Ctrl+Shift+P` 打开命令面板
4. 输入 `Extensions: Install from VSIX...`
5. 选择下载的 `.vsix` 文件

### 从市场安装（发布后）
1. 打开 VS Code 扩展面板 (`Ctrl+Shift+X`)
2. 搜索 `Encoding Converter`
3. 点击安装

## 开发调试

### 环境要求
- Node.js >= 14.x
- VS Code >= 1.80.0

### 安装依赖
```bash
cd encoding-converter
npm install
```

### 编译项目
```bash
npm run compile
```

### 调试插件
1. 在 VS Code 中打开项目文件夹
2. 按 `F5` 启动扩展开发主机
3. 在新窗口中测试插件功能

### 打包插件
```bash
npm run package
```

生成的 `.vsix` 文件可用于分发和安装。

## 技术架构

本插件使用以下核心库：
- **jschardet**：用于自动检测文件编码
- **iconv-lite**：用于编码转换

## 注意事项

- ⚠️ 转换操作会直接修改原文件，建议转换前做好备份
- 💡 如果自动检测的编码不正确，可以手动选择其他编码尝试
- 📝 转换后需要重新打开文件才能看到正确的内容
